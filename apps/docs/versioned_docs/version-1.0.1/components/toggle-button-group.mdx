## Usage

<Tables
  hideHeader
  headers={['title', 'description']}
  args={[
    {
      title: 'Usage and conditions (for acceptance criteria)',

<div>
  <p>Button groups serve as a structured approach to organizing buttons, grouping them according to related actions. This organizational method enhances user experience and interface clarity.</p>
  <p>A button group comprises multiple instances of our standard button component, ensuring consistency in design and functionality across the group. These groups inherit the established button designs and states, maintaining visual coherence throughout the interface.</p>
  <p>The utilization of button groups presents distinct advantages over haphazardly placing buttons in close proximity:</p>
  <ol>
  <li><b>Comprehensive Action Sets:</b> By grouping buttons, you can display a cohesive set of actions alongside additional individual buttons. This arrangement streamlines user interaction by presenting related actions together, reducing cognitive load and enhancing efficiency.</li>
  <li><b>Efficient Organization:</b> Button groups are instrumental in organizing various sets of buttons. Whether categorizing actions by function or grouping similar tasks together, this approach facilitates intuitive navigation and enhances the overall usability of the interface.</li>
  <li><b>Space Optimization:</b> In scenarios where space is limited, such as mobile interfaces or compact layouts, button groups offer a space-efficient solution. By consolidating closely linked buttons within a single group, horizontal space is conserved without sacrificing functionality. This ensures that the interface remains clean and uncluttered, promoting ease of use and visual appeal.</li>
  </ol>
  <img src="/img/toggle_button_group.png" />
  <strong>Usage</strong>
  <p>Arrange toggle buttons with similar functions into button groups, ensuring that their grouping is based on function rather than mere proximity.</p>
  <p>Generally, it's advisable to utilize default button types within these groups. However, in rare cases, incorporating a primary button is acceptable, but limit it to one per group, if used at all.</p>
  <p>Maintain a reasonable number of buttons in each group to prevent overwhelming users.</p>
  <p>Avoid using toggle button groups to indicate selections or as replacements for tabbed navigation.</p>
  <p>It's not recommended to mix text-labeled buttons with icon-only buttons, except when pairing a text-labeled button with an icon-only button featuring a downward triangle. This icon indicates a dropdown menu offering actions associated with the button.</p>
</div>)
