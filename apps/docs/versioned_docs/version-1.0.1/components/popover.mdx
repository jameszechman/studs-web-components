### Interaction

<b>Behavior and states</b>
<p>Popover is opened by pressing a trigger, and closed by pressing the trigger again or pressing the dismiss icon. Upon press, the component hovers in one of four positions; to the top, right, bottom, or left of the trigger. On small screens, it may cover the entire viewable area including the trigger.</p>
<p>Interactive elements within the content of an open popover can include a scroll bar, external links, a <a href="https://www.delldesignsystem.com/components/view-more-less/" target="_blank">view more/less</a> component, a <a href="table" target="_blank">table</a> component, and other displayed content but should not include submittable forms, select components, or other complex interactions. Consider <a href="drawer" target="_blank">drawer</a> or <a href="modal" target="_blank">modal</a> for more extensive supplemental content.</p>

## Additional Guidance

<Tables
  hideHeader
  headers={['title', 'description']}
  args={[
    {
      title: 'Content',
      description: (
        <div>
          <ul>
            <li>
              <b>Popover title</b>
              <p>Be brief and descriptive about the purpose of the popover. The length should not exceed one line.</p>
              <p><strong>Do:</strong> Help me choose</p>
              <p><strong>Don't:</strong> Comparing Pentium-class versus Xeon-class processors</p>
              <p><strong>Body text</strong></p>
              <p>Popover content should be centered around the trigger and what is essential for a user to know in relation to it. Content can include imagery, tables, reference links, and more text than a tooltip, but it should not require progressive actions like inputs and buttons. If content exceeds the container height, a scrollbar will appear, but consider placing longer content within the on-page experience or using a different component like drawer.</p>
              <p>Avoid placing complex content or functionality inside a popover. Content should be short and limited to simple, contextual functionality that relates directly to a particular item or control on the page. More complex content, such as a form, should be placed in either a modal or on the page directly.</p>
              <p><strong>Capitalization</strong></p>
              <p>Use sentence case for labels, default values, detected values, and autocompleted text, unless referencing product names and proper nouns.</p>
            </li>
          </ul>
        </div>
