        <div>
          <p>Tabs help organize similar types of content on a screen and allow users to move easily between alternate views within the same context.</p>
          <strong>Labels</strong>
          <p>Tab labels should be brief. Use one to two words for the tab label. The labels should also accurately describe the content associated with each tab.</p>
          <strong>Grouping</strong>
          <p>Organize content clearly within tabs, ensuring that the information in each tab is intuitive and predictable. If users require simultaneous access to all content across tabs, consider implementing a different navigation solution.</p>
          <strong>Order</strong>
          <p>Tab order should be consistently applied throughout the interface. The default or selected tab should display the primary or most essential information first. Tabs containing related content should be positioned adjacent to each other.</p>
          <strong>Stacking</strong>
          <p>Use responsive, stacked tabs for contained tab scenarios, with three or more tabs visible on extra-small and small screens. Stacked tabs are ideal for complex content.</p>
          <p>Tabs are links include a selected state indicator at the bottom of the selected tab label.</p>
          <ol>
          <li>Selected tab: includes a Orange 50 tab indicator at the bottom of the tab and label as well as a fill of Grey 10</li>

          <li>First state tab: includes a Gray 70 divider above the first tab.</li>
          <li>Tab label: a concise title representing the tab’s content.</li>

## Additional Guidance

<Tables
  hideHeader
  headers={['title', 'description']}
  args={[
    {
      title: 'Content',
      description: (
        <div>
          <ul>
            <li>
              <b>Capitalization and case</b>
              <p>
                Use sentence case (capitalize only the first word) unless
                referring to a proper noun.
              </p>
            </li>
            <li>
              <b>Punctuation</b>
              <p>Do not punctuate tab labels.</p>
            </li>
            <li>
              <b>Tab labels</b>
              <p>
                Write short tab labels and use plain language. Try to keep the
                label text to one or two words. Short labels are more scannable.
                If you need longer labels, it may be a sign that the categories
                are too complicated for a tab control. Keep in mind that the
                character length of a label will affect the width of the tab,
                and potentially cause other tabs to be truncated, only appearing
                if the tab “more” is pressed.
              </p>
            </li>
          </ul>
        </div>
      ),
    },
