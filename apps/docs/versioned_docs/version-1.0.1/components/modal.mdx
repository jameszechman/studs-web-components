Modals are used to display content that temporarily blocks interactions with the main view of an application. Modals should be used sparingly only when necessary.

## Usage

  hideHeader
  headers={['title', 'description']}

<div>
  <p>
    Modals present essential information that necessitates user acknowledgment before proceeding. They overlay the interface, pausing other interactions until the user takes the necessary action. Employ them when the user's decision is crucial.
  </p>
  <p>
    <strong>Modal Components</strong>
    <ol>
      <li><b>Header:</b> Optional title.</li>
       <li><b>Body:</b> The core message.</li>
        <li><b>Dismiss Icon:</b> Allows users to close the modal.</li>
         <li><b>Call-to-Action:</b> An optional set of buttons.</li>
    </ol>
  </p>

  <b>
    Interaction Interruption
  </b>
  <p>Modals serve to momentarily halt the user's progression. Their primary role is for tasks demanding immediate attention or acknowledgment, not just passive alerts. When active, an overlay conceals the webpage, ensuring users focus on the modal and do not revert to their previous task unless they complete the modal task or dismiss it.</p>
  <b>
    Modal Presentation
  </b>
  <p>Stick to one modal at any moment. Avoid stacking modals or introducing a new modal without user input.</p>
  <b>
    Task Clarity  
  </b>
  <p>Ensure the modal provides concise instructions on the required action or how to dismiss it. Clarity is paramount.</p>
  <p>
    <strong>Button Placement</strong>
  </p>
  <p>The modal's size dictates the button alignment. Generally, they should be at the modal's base. In most scenarios, align buttons to the right.</p>
  <b>
    Content Management
  </b>
  <p>Should content surpass the modal's height, a vertical scrollbar emerges. However, for optimal user experience, avoid making users scroll. Ensure modals never require horizontal scrolling. Opt for a larger modal if necessary.</p>
</div>)
    },
    {
      title: 'Style',
      description: (<div>
        <strong>Modal overlay</strong>
        <p>A modal contains a transparent background overlay which partially obscures the content behind the modal. The overlay helps interrupt the on-page experience, so users can focus on the tasks outlined in the modal. The overlay should cover the page content and prevent the user from interacting with it until the action is complete or the modal is dismissed.</p>

 ## Writing Guidance
<div>
  <h3>Use Modals for Critical Information</h3>
  <p>Modals should be used to deliver key information at pivotal moments. Used correctly, they will clearly articulate user actions that are required to move forward. Don’t use modals for non-critical information.</p>
  <div class='exampleImages'>
    <div class='wrapper -do'>
      <p class='-do'>Do</p>
      <figure>

      </figure>
    </div>

    <div class='wrapper -dont'>
      <p class='-dont'>Don't</p>

        <figcaption></figcaption>
      </figure>
    </div>
  </div>

  <h3>Keep Modals Concise</h3>
  <p>Communicate essential information without overwhelming the user. A concise modals provided enough context to inform the user and prompt the necessary action. Too much detail can lead to confusion and user fatigue.</p>
  <div class='exampleImages'>
    <div class='wrapper -do'>
      <p class='-do'>Do</p>

    </div>
  </div>

  <h3>Provide Clear Actions</h3>
  <p>To aid in scanning and clarity, align the provided actions in buttons with the title of your modal. Actions should also be specific, avoiding generic labels don’t inform users about what will happen next.</p>

        <div>
          <b>Body text</b>
          <p>
            Modal content should be centered around task-oriented scenarios and
            what a user must do to complete that task. If the task is
            complicated or requires more than a few lines of instructional text,
            consider using another component or experience, such as a separate
            page in the workflow. If content exceeds the container height, an
            overflow scrollbar will appear, which may cause users to miss
            important context.
          </p>
          <b>Capitalization</b>
          <p>
            Use title case in headers and sentence case for body text, unless referring to a
            proper noun. For buttons, use title case.
          </p>
          <b>Modal headers</b>
          <p>
            For modal headers, be brief and descriptive about the action a user
            must take to resolve the task. Try to use verb and noun combinations
            that clearly describe the dialog’s task or purpose. Header length
            should not exceed one line.
          </p>
          <b>Punctuation</b>
          <p>
            Do not punctuate button text or header text. For the modal body, use
            normal punctuation rules.
          </p>
