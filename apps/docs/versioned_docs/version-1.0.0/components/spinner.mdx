Spinners are used to show the progress of an indeterminate operation.

<Tables
  hideHeader
  headers={['title', 'description']}
  args={[
    {
      title: 'Usage and conditions (for acceptance criteria)',
      description: (
<div>
 <p>Spinners inform users that their action is currently in process. For indicating loading periods, spinners should be reserved for content that cannot be depicted through skeleton loading components, such as data charts or cards.</p>
 <p>Spinners indicate that a system process is going on that will end with content being displayed to the user. They animate as soon as user action is initiated and disappear once the content appears. Only show a spinner if the expected wait time is more than a second. There should only be a single spinner on a page at any time.</p>
  
 <strong>Anatomy</strong>
 <ol> 
 <li><b>Track:</b> Background that the segment rotates on during the looped animation.</li>
 <li><b>Segment:</b> Animated element of the spinner that simulates rotation.</li>
 </ol>
 <img src="/img/spinner_anatomy.png" />
  <strong>When to Use</strong>
  <p>Utilize a spinner to indicate ongoing loading processes. In certain cases, the user interface may update prior to data being fully saved. Employ a spinner to convey ongoing background activity until information is successfully stored.</p>
  <strong>Sizes</strong>
  <ul>
  <li><b>Extra Small:</b> 16px, used with text labels or inside other UI elements, like a button.</li>
  <li><b>Small:</b> 22px, used within larger visual elements, like merge widget status icons.</li>
  <li><b>Medium:</b> 38px, indicates larger components are loading, like a card.</li>
  <li><b>Large:</b> 48px, indicates a large content area of the UI is loading.</li>
  </ul>
