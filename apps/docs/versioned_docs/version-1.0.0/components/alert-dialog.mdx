<div>
  <p>
   An alert dialog is a nonmodal, passive dialog that keeps users informed without blocking access to the base page. The user can continue with the experience, regardless if the alert is still on screen. Use for interactions that aren’t critical.
  </p>
  <ul>
    <li>
     Alert message body length should be concise and not exceed more than two lines.
    </li>
    <li>
    Titles should be descriptive, containing two to three words to explain the context of the message.
    </li>
    <li>Alerts should be related to the user’s task.</li>
    <li>This type of feedback should be timely and inform users when the message was delivered or by when an action needs to occur.</li>
    <li>If there is an action, give users the next steps to address the notification.</li>
    <li>Limit sending notifications that aren’t necessary to prevent alert fatigue.</li>
    <li>
      Avoid sending duplicate notifications, even if the user hasn’t interacted or dismissed the original message.
    </li>
    <li>Notifications should appear in chronological order.</li>
  </ul>

## Additional Guidance

<Tables
  hideHeader
  headers={['title', 'description']}
  args={[
    {
      title: 'Accessibility Concerns',
      description: (
        <div>
          <ul>
            <li>
              <b>
              Alert titles must accurately describe the meaning and purpose of the alert dialog
              </b>
              <div>
                The purpose of the alert should be clearly understood by reading the title alone.
              </div>
              <div>
                WCAG 2.1:&nbsp;
                <a href="https://www.w3.org/WAI/WCAG21/Understanding/headings-and-labels.html">
                  2.4.6 Headings and Labels
                </a>
              </div>
            </li>
                      <li>
              <b>
               Avoid timed alerts
              </b>
              <div>
                Those using assistive technology or who have cognitive difficulties may miss or find it hard to react to a timed message before it disappears.
              </div>
              <div>
                WCAG 2.1(AAA, optional):&nbsp;
                <a href="https://www.w3.org/WAI/WCAG21/Understanding/no-timing.html">
                  2.2.3 No Timing
                </a>
              </div>
            </li>
                      <li>
              <b>
                Provide users with settings to turn off nonessential alerts or change the frequency
              </b>
              <div>
                Non-urgent notifications can be distracting for people with cognitive disabilities and can be hard to see for those with partial or low vision, since notifications typically display on a non-disruptive area of the screen.
              </div>
              <div>
                WCAG 2.1 (AAA, optional):&nbsp;
                <a href="https://www.w3.org/WAI/WCAG21/Understanding/interruptions.html">
                  2.2.4 Interruptions
                </a>
              </div>
            </li>
                     <li>
              <b>
                Avoid using icons alone to convey meaning
              </b>
              <div>
                Icons should be used to reinforce the meaning, not explain it.
              </div>
            </li>
            <li>
              <b>
                Make sure icons are universally recognizable
              </b>
              <div>
                If there is any doubt about the meaning of an icon, make sure heading text accurately describes the purpose of the message along with the icon.
              </div>
            </li>
          </ul>
        </div>
      ),
    },
  ]}
/>
