      <p>
        Pagination divides a collection of data or other content types such as images, cards, or search results into multiple pages showing a fixed number of items per page.
      </p>
      <p>
        Consider using the pagination component to make a large amount of related content more consumable to your users, or to optimize on-page real estate or load times in your experience. Pagination elements allow users to control how much information they’d like to see per page, and navigate all of the data or content types in a smaller, more controlled view. Controls are centered below the paginated items, whether it is on-page, below a table, or embedded in another component.
      </p>
      <b>On-page</b>
      <p>
        Pagination is typically not required when there are 25 or fewer items per
        page. Depending on the context, however, consider reducing the capacity of
        items per page. Considerations include smaller screen sizes or when other
        variables increase vertical scroll.
      </p>
      <b>
        Embedded
      </b>
      <p>Pagination can be used within components, such as tables, to help users navigate large data sets or other content.</p>
</div>)
    },

    {
      title: 'Style',
      description: (<div>
        <p>The pagination component is comprised of the following elements:</p>
        <ol>
          <li>Label of total pages: shows the number of pages based on the capacity of items per page.</li>
          <li>Previous button: takes the user to the previous page.</li>
          <li>Page button: takes the user to the specific page of results</li>
          <li>Ellipses: Used to truncate the results if there are over the number of spaced page buttons</li>
          <li>Current page: Displays the current page that the user is viewing.</li>
          <li>Next button: takes the user to the next page.</li>
          <li>Capacity control: select component that allows the user to determine the number of items per page. The default values are 10, 25, 50, 100, and 250, but can be customized.</li>
          <li>Location: text input showing what page the user is viewing, which can also be used for navigation.
