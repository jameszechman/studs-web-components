Button groups can be used to group related `studs-buttons` into sections. 
## Usage

<Tables
  hideHeader
  headers={['title', 'description']}
  args={[
    {
      title: 'Usage and conditions (for acceptance criteria)',
      description: (
<div>
  <p>
   Buttons can be organized into groups, with each group acting as a collection of related actions. A button group consists of multiple instances of our standard button component, and it supports the inherited button designs and states.
  </p>
<p>
Utilizing a button group to organize buttons offers advantages over merely placing buttons in close proximity, especially for:
</p>

  <ul>
    <li>
Displaying a set of buttons alongside additional individual buttons.
    </li>
    <li>
Organizing various button sets together.
    </li>
    <li>Conserving horizontal space when presenting a series of closely linked buttons.</li>
  </ul>
   <img src="/img/button_group.png" />
<p><b>Usage</b></p>
<p>Organize buttons with similar functions into button groups, but don't cluster them together just because of their proximity.</p>
<p>Typically, you should stick to using only default button types within button groups. In exceptional situations, you may include a primary button in these groups, but limit it to just one primary button per group, if used at all.</p>
<p>Keep the number of buttons in a group manageable to avoid overwhelming users.</p>
<p>Button groups should not be used to denote selections.</p>
<p>They also shouldn't serve as substitutes for tabbed navigation.</p>
<p>Mixing text-labeled buttons with icon-only buttons is generally discouraged, except when pairing a text-labeled button with an icon-only button featuring a downward triangle, which indicates a dropdown menu offering actions related to the button.</p>
</div>
)
    },
    {
      title: 'Behavior: Active, Focus, Hover, Disabled, Touch Start, Touch End',
      description: <div>
      <p>Behavior inherits button actions, style and variations.</p>
      <p>Button groups are limited to medium (the default size) and small sizes only.</p>
      </div>
    },
]}
