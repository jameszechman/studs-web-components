      <ol>

      <li>Handle</li>

      </ol>
      <strong>Track</strong>
      <p>The track shows the full range of values that can be selected on the slider. It has two sections: active and inactive.</p>
      <ul>
      <li>The <b>active</b> section of the track is from the minimum value to the handle. For range selection sliders, the active track is between the two handles.</li>
      <li>The <b>inactive</b> section of the track is from the handle to the maximum value, or outside the two handles of a range selection slider.</li>
      </ul>
      <p>For left-to-right (LTR) languages, the values increase from left to right. For right-to-left (RTL) languages, this is reversed.</p>
      <strong>Stop indicators</strong>
      <p>On discrete sliders, stop indicators show the predetermined values that can be chosen along the track. The slider snaps to the closest stop. Avoid having too many stop indicators on a slider, because it can become visually crowded and difficult to adjust the value.</p>
      <p>On all other types of sliders, stop indicators should be used to show the end values of inactive tracks.</p>
      <strong>Handle</strong>
      <p>The handle can be moved along the track to choose a value.</p>
      <p>When sliders have two handles, the handles choose the minimum and maximum values in a range.</p>
      <strong>Value (optional)</strong>
      <p>The value displays the specific value that corresponds with the handle’s placement. </p>
      <p>A value appears when interacting with the corresponding handle. For range sliders, only one value is shown at a time.</p>
      <strong>Adding extra elements to the slider</strong>
      <p>Instead of showing the built-in value label, a separate text input field can be added outside of the slider. If this is added, the slider and value in this text field should automatically update to match each other.</p>
      <p>Make sure people can tab to the text field directly after the slider.</p>
      <p>Icons or text can be added outside the slider to indicate the range of values and make the slider range more accessible. This can be used instead of a stop indicator.</p>
      <strong>Select and drag</strong>
      <p>Select a value by dragging the handle.</p>
      <strong>Select jump</strong>
      <p>Select a value by selecting part of the track.</p>
      <strong>Select and arrow</strong>
      <p>Select a value by keyboard navigation.</p>
      <p><b>Tab:</b> Focus lands on handle</p>
      <p><b>Arrows:</b> Selected value increases or decreases by one value or stop indicator</p>
      <p><b>Space + Arrows:</b> Selected value increases or decreases by a larger interval or stop indicator</p>
      </div>

        <strong>Touch</strong>
        <p>When tapped or dragged, the handle width shrinks to provide interaction feedback, and the value appears.</p>
        <strong>Cursor</strong>
        <p>When hovered, the cursor changes. When clicked, the handle width shrinks, and the value appears.</p>
        <strong>Focus and navigation</strong>
        <p>Initial focus lands directly on the handle, since it’s the primary interactive element of the slider.</p>
        <p>The slider value can then be adjusted using the arrow keys or other keyboard navigation options.</p>
        <strong>Color contrast</strong>
        <p>Use visual anchors to ensure the slider meets accessibility standards.</p>
        <p>If the inactive track of a continuous slider has less than a 3:1 contrast with its background, add visual indicators to identify the ends of the inactive track. These can be stop indicators, or icons with at least 3:1 contrast.</p>
        <strong>Labeling elements</strong>
        <p>The accessibility label for a slider is typically the same as the slider's adjacent text label. It should have the <b>slider</b> role.</p>
        <p>If the UI text is correctly linked to the slider, assistive tech (such as a screenreader) will read the UI text followed by the component’s role.</p>
              </div>)
